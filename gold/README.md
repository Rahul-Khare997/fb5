# gold/: the exemplar library

Rules transfer correctness; examples transfer taste. This folder holds work that actually
cleared the bar — pieces the taste gate passed with distinction, or past work the team
holds up as "this is us at our best". Phase 0 reads the exemplar matching the task's
domain before producing, and imitates its STRUCTURE and TECHNIQUE, never its content: the
kill test applies doubly to anything sitting next to a gold example.

## Format

One file per exemplar at `gold/<domain>/<short-name>.md` (domains = the 21 craft names).
Non-text work (a page, a video) is a file with the link or path plus the header. Every
file starts with exactly three header lines:

```
IMITATE: <the structural moves worth stealing — e.g. "opens with the buyer's own words;
  one metric per section, each recomputed; ends on the unresolved risk">
IGNORE: <what is incidental and must not be copied — the topic, the names, the numbers,
  anything client-specific>
ADDED: <date, and by what — e.g. "2026-07-07, gate run on the pricing page">
```

Then the work itself (or the link).

## What gets in

- The gate passed it with zero findings AND flagged it as signing-bar work, and the
  operator agreed. The gate offers; a human admits.
- Or the team nominates a past piece during the tailor interview (docs/TAILOR.md).

## What gets out

Same garden rule as the standards: an exemplar the team has outgrown, or one whose moves
have become the house default anyway, gets deleted. Three strong exemplars per domain beat
ten mixed ones; past three, replace the weakest instead of adding. An empty domain folder
is honest — it means no work has earned it yet.

## Why structure, never content

Copying content produces yesterday's deliverable with today's nouns; the kill test kills
it. Copying structure (the argument shape, the layout logic, the pacing) is how taste
actually transfers between pieces — it is also what a strong model does when it "has
taste": it reaches for moves, not sentences.
